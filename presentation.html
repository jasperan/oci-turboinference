oci-turboinference

Run 400B+ parameter models on a $1/hr GPU

Models keep getting bigger. GPUs don't.

  • GPT-4: ~1.8T params  |  DeepSeek-V3: 671B  |  Llama 70B: needs 140GB at fp16
  • A10 GPU: 24GB VRAM. That's it.
  • Most people can't afford A100s or H100s

What if the model doesn't need to fit?

  • Extreme quantization: compress 27B params into 8.6GB (2-bit)
  • CPU offloading: spill layers to 240GB system RAM
  • Smart layer splitting: GPU handles hot layers, CPU handles the rest

Why smaller = faster

Less VRAM pressure = more bandwidth for compute

  • Q4_K_M uses 22.5GB of 23GB total. The GPU is memory-starved.
  • IQ2_XXS uses 13.4GB leaving 10GB free for KV cache + compute.
  • Memory bandwidth is the bottleneck, not raw compute.
  • Result: 24% faster generation despite lower precision.
